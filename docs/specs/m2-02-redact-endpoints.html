Spec M2-02 — /redact + /unredact

Second M2 spec (plan). Adds the two redaction endpoints to the M2-01 FastAPI app, wrapping prompt_redact_core, with the flat-v1 token-map wire contract and the agreed error mapping.

File: prompt_redact_service/app.py (models + handlers). Tests: tests/test_service_redact.py. No new deps; runs without Presidio via an injected fake analyzer.

Endpoints

  • POST /redact — {text, token_map?, language?} → {redacted_text, token_map}.
  • POST /unredact — {text, token_map} → {text}.

Token-map wire contract (v1)

Flat {token: original} JSON — identical to the in-memory shape prompt_redact_core uses, so there is no transform at the boundary. Documented as v1; versioning is added only on a breaking change (M2 decision).

Error mapping

ConditionStatus
Malformed body (missing/typed fields)400
Token-shaped substring in /redact input (T5)400
Invalid token_map / unsupported language400
Unknown token in /unredact (core raises UnknownTokenError)422
Service not ready (model not loaded)503
Unexpected engine failure500 (FastAPI default)

Resolved decisions

  • Validation errors → 400, not FastAPI's default 422. The RequestValidationError handler is overridden so a malformed body is a 400 (matching the plan), leaving 422 to mean exactly one thing: a valid request whose token map is out of sync (unknown token).
  • Error responses are generic — no input echoed. Handlers return fixed messages (e.g. “malformed request body”), never str(exc) or the offending value. Pydantic's default validation detail and some core error messages can contain the submitted text or originals; echoing them would leak PII (threat T2). Status code + generic message only.
  • Per-call language is validated against the loaded model. The service runs one eagerly-built analyzer (one language). A request whose language differs from the analyzer's configured language is rejected 400 rather than silently mis-detected. (Only the language code — not PII — appears in that message.)
  • Readiness gate. Both endpoints return 503 if the analyzer never built (consistent with /healthz).

Deferred to M2-03

The request size cap (413, T9) and logging discipline (no request/response bodies in logs; scrub the unexpected-500 path so a traceback can't carry input text — T2/T3). Until then, rely on FastAPI's default 500 (which doesn't echo the body in the response) and avoid logging bodies.

Test plan (tests/test_service_redact.py)

  • /redact: single detection; no-PII identity; reuse of a supplied token_map (cross-call numbering); token-shaped input → 400; unsupported language → 400; malformed body → 400; not-ready → 503.
  • /unredact: restores originals; unknown token → 422; no-token passthrough; malformed body → 400.
  • Round trip through the API: redact then unredact reconstructs the original.
  • All via an injected fake analyzer (with config.language + preset detections) + TestClient — no Presidio.

Definition of done

  • Both endpoints implemented with the flat-v1 contract and the error mapping above.
  • Validation → 400; unknown-token → 422; no input echoed in any error.
  • Tests green without Presidio; full suite green.
