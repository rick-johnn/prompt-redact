Spec M2-03 — Request size cap + logging discipline

Third M2 spec (plan). Hardens the service against two threats deferred from M2-02: oversized bodies (T9) and identifier leakage through logs / crash paths (T2/T3).

File: prompt_redact_service/app.py. Tests: tests/test_service_limits.py. No new deps; no Presidio needed.

Request size cap (T9)

  • BodySizeLimitMiddleware (pure ASGI) rejects bodies over a configurable cap with 413 {"detail":"request body too large"}.
  • Two layers: a fast reject when a declared Content-Length exceeds the cap (so a huge declared body is refused before it's read), and a running byte count while buffering the body — covering chunked or missing Content-Length, so an oversized stream can't slip through to the expensive NER pass. Buffering is bounded by the cap.
  • Configurable via max_body_bytes (default: the PROMPT_REDACT_MAX_BODY_BYTES env var, else 1 MB).
  • /healthz and other body-less requests pass through unaffected.

Logging discipline & the scrubbed 500 path (T2/T3)

  • No bodies in logs. The service logs nothing containing request/response bodies; FastAPI/uvicorn access logs carry method/path/status only. Our own log lines never include text or the token map.
  • Scrubbed unexpected-error path. Each handler wraps the core call in a final except Exception that logs only the exception type name and raises HTTPException(500, "internal error") with from None. Dropping the exception chain means the original error — whose message could embed the input text (e.g. an engine error like “choked on <text>”) — is never re-raised to the server-error middleware, so no traceback or message carrying input is logged or returned.
  • This is why the unexpected-error response is a flat {"detail":"internal error"} and contains none of the input.

Test plan (tests/test_service_limits.py)

  • Oversized body → 413 with the generic detail; body within the cap → 200.
  • Cap honored from the PROMPT_REDACT_MAX_BODY_BYTES env var.
  • /healthz not blocked by the cap.
  • An engine error whose message embeds the input → 500 {"detail":"internal error"}, and the input string is asserted absent from the response.

Definition of done

  • Configurable body-size cap returning 413; enforced by declared length and actual bytes.
  • Unexpected errors return a generic, input-free 500; only the exception type is logged.
  • Tests green without Presidio; full suite green.
