Spec M1-10 — Real-world / independent recall

Follow-up to the M1-08 synthetic gate, prompted by the architecture review: the synthetic corpus and the recognizers share authorship, so its 1.000 recall proves the pipeline is consistent and regression-free but says little about real-world recall. This adds an independent-corpus harness that measures the analyzer against externally-authored, messy/adversarial data.

Module: evals/realworld.py. Sample: evals/corpus/realworld_sample.jsonl. Tests: tests/test_realworld.py.

How it differs from the gate

  • Loader, not generator. It reads an external JSONL corpus (same Example format as M1-07) authored independently of the recognizers — breaking the test/code coupling.
  • Report-only. Real-world recall is informational, not a pass/fail gate — it never fails a build. It reuses the exact scoring (run_eval.evaluate / metrics); only the corpus source differs.
  • PII boundary. A real hand-labeled corpus may contain real identifiers — keep such files OUT of the repo (gitignored evals/corpus/real/); only the synthetic, adversarial sample is committed.

The committed sample

A hand-authored, deliberately messy/adversarial synthetic corpus (20 examples, 25 gold spans, 2 clean) — written to be realistic and hard, not to match the detectors: misspelled / lowercase / non-Western / accented / month-like names, odd phone formats, space-separated SSN, full street addresses, an obfuscated email, no-context MRN / member / Rx IDs, valid NPI/DEA/credit-card, plus a non-PII 10-digit number. It still shares one author with the code, so it is a step toward independence, not the full thing — a truly independent number needs a corpus hand-labeled by someone who never saw the recognizers.

Leakage 0.15 (3/20). Reading: the synthetic gate's uniform 1.000 was optimistic. On adversarial input, real gaps surface — obfuscated emails, non-standard SSN spacing, street-level addresses — while PERSON proves robust. Caveat: small n (several types have a single example), so these are illustrative, not statistically robust; the value is in the gaps surfaced, not the exact fractions.

Usage

python -m evals.realworld <corpus.jsonl> [spacy_model]
# e.g. the committed adversarial sample:
python -m evals.realworld evals/corpus/realworld_sample.jsonl
