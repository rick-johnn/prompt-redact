Spec M1-06 — Redactor

Sixth implementation spec for M1. Implements redact(text, token_map) -> (redacted_text, token_map): the top-level entry point that ties the already-built pieces together. It is the reverse of unredact.

Module: prompt_redact_core/redactor.py. No new errors. Maps to the plan's redact() scope item and the test_redactor.py test file.

What it composes

  • Spec M1-02 — assert_no_token_shapes (the T5 input precondition).
  • Spec M1-04 — RedactionAnalyzer.analyze (PII detection → non-overlapping Detections).
  • Spec M1-01 — assign_tokens (caller-map merge) and apply_replacements (right-to-left splice).

By M1-06 every part already exists and is tested; this spec is thin orchestration plus the end-to-end tests that prove the parts compose.

Algorithm

  1. Guard. assert_no_token_shapes(text) — reject input that already contains a [TYPE_N] shape (raises TokenShapedInputError) before any analysis. Running first avoids paying for detection on input we will reject, and keeps the minted-token vocabulary unambiguous (threat T5).
  2. Detect. detections = analyzer.analyze(text).
  3. Assign. replacements, new_map = assign_tokens(detections, token_map or {}) — known originals reuse their token, new ones mint max_N + 1 for their type. The input map is copied, never mutated.
  4. Replace. apply_replacements(text, replacements) — single pass, right-to-left.

Resolved behaviour decisions

  • The analyzer is injected, not constructed. redact takes analyzer as a required keyword-only argument rather than building one internally. The analyzer owns the spaCy model (hundreds of MB, seconds to load); a fresh one per call would reload it. The caller — and in M2 the service, once at startup — builds one RedactionAnalyzer and reuses it. Any object exposing analyze(text) -> list[Detection] satisfies the contract, which is also how the tests inject a fake. (Decision 2026-06-05; a convenience default that lazily builds and caches an analyzer was rejected to avoid hidden global state and per-config surprises.)
  • Guard before detect. The T5 guard runs first, so token-shaped input is rejected without invoking the analyzer at all.
  • Return shape: a plain (str, dict) tuple. Matches assign_tokens' tuple return and the plan's redact(...) -> (redacted_text, token_map). The returned map is a fresh dict the caller owns and round-trips (ADR 0002).
  • No PII / empty text. Falls out of the algorithm: no detections → no replacements → text returned unchanged, with a copy of the supplied map.
  • Token-map contract. In-memory flat token -> original dict, same as the rest of M1. The wire/versioning shape stays deferred to M2 (M0 decision).

Round-trip property

The headline property unredact(redact(text)) == text was already proven at the token-pipeline level in Spec M1-03 using synthetic detections. M1-06 closes it for real by adding detection on top: an integration test redacts a sample with the live analyzer and asserts the round trip, that the raw identifiers are absent from the redacted text, and that at least one token was minted.

Public surface (redactor.py)

  • redact(text, token_map=None, *, analyzer) -> tuple[str, dict[str, str]] — redact PII; raises TokenShapedInputError (T5) on token-shaped input. Exported from the package as prompt_redact_core.redact.

No new module-level state and no Presidio import in redactor.py itself — Presidio arrives only through whatever analyzer the caller passes.

Test plan (tests/test_redactor.py)

  • Without Presidio (fake analyzer): single detection; multiple distinct identifiers → distinct tokens; repeated identifier in one call → same token; no detections → identity + empty map; empty text; detections supplied out of offset order still number by offset.
  • Cross-call continuity: passing the returned map back in reuses tokens for known originals and mints the next number for new ones.
  • Map immutability: the supplied map object is not mutated and a fresh dict is returned.
  • T5: token-shaped input raises TokenShapedInputError and the analyzer is never called.
  • Round trip (fake): unredact(redact(text)) == text.
  • Integration (real Presidio, skipped if unavailable): redact a sample, assert round trip, that raw identifiers are gone, and that a token was minted.
