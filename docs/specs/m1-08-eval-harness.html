Spec M1-08 — Eval harness

Eighth and final implementation spec for M1. Scores the analyzer's detection against the gold corpus (Spec M1-07) and acts as the regression gate that defines M1's exit.

Package: evals/ — metrics.py (pure scoring), run_eval.py (CLI + gate). Tests: tests/test_eval_harness.py. Scoring and the eval loop are pure (analyzer injected); only the real run needs Presidio.

What it measures

  • Per-entity recall (bucketed by gold type) and per-entity precision (bucketed by predicted type).
  • Leakage rate — fraction of examples where any gold span was not caught.
  • Matching is exact character offset: a partial match (caught "Doe", not "John Doe") is a miss, because the visible remainder leaks. Recall is type-agnostic for catching — what matters is that the characters were redacted, whatever the detector labelled them; precision is bucketed by the predicted label.
  • Recall is measured on the analyzer's detections, which is exactly what the redactor redacts (it splices a token over every detected span) — so detection recall is redaction recall, and the harness needn't re-run replacement.

Resolved decisions (2026-06-05)

  • Gate scope: implemented recognizer types only. Recall ≥ 0.99 gates PERSON, EMAIL_ADDRESS, PHONE_NUMBER, US_SSN, CREDIT_CARD, DATE_TIME, LOCATION, NPI, DEA. MRN, MEMBER_ID, RX_NUMBER are reported but not gated — their recognizers are deferred (Spec M1-05); gating them would fail by construction. They appear in the report marked report so the gap is visible, and the corpus already carries them as gold so they can be gated once their recognizers land and are tuned.
  • Leakage: reported, not gated. The M0 leakage bar (≤ 1/10,000) is only statistically meaningful at tens of thousands of inputs; M1 gates on per-entity recall (the plan's stated exit) and prints leakage at the run's corpus size as informational. Strict leakage-gating at scale is a later step.
  • Analyzer injected. evaluate(corpus, analyzer) takes any object with analyze(text), so the entire harness loop is unit-testable with a fake analyzer; only main constructs the real RedactionAnalyzer.

Gate & exit code

python -m evals.run_eval [n_per_template] [seed] generates the corpus, runs the analyzer, prints the report, and exits non-zero if any gated type is below target — so it drops straight into CI. Report.passed() encodes the gate; Report.gate_failures() lists the offenders.

Public surface (evals/)

  • score_corpus(pairs) -> Report — pure aggregation over (gold, predicted) span pairs.
  • evaluate(corpus, analyzer) -> Report — runs the analyzer over the corpus and scores.
  • Report — recall(), precision(), leakage_rate, gate_failures(), passed(), format().
  • RECALL_TARGET, GATED_TYPES.

Test plan (tests/test_eval_harness.py)

  • Scoring: perfect detection; a missed span (recall miss + leak); a false positive (precision miss, no leak); partial-offset = miss; type-agnostic catching; no-gold example never leaks; empty corpus.
  • Gate: a gated type below 0.99 fails; an ungated type (MRN) below target does not; a gated type at target passes; report formats.
  • evaluate(): with a fake analyzer — perfect detection passes; a blind analyzer fails.
  • Integration (real Presidio, skipped if unavailable): the harness runs end-to-end over the generated corpus and returns a populated report.

M1 exit status

This spec completes the M1 machinery: redactor + recognizers + corpus + a gating harness, all unit-green. M1 formally exits only when run_eval is executed in a Presidio-enabled environment and the recall gate passes (and the threshold is tuned if any gated type falls short). That run can't happen in the design sandbox (no Presidio), so it is the first action on a dev/CI machine. Until then, recall numbers are unverified and the gate result is pending.

Definition of done

  • metrics.py + run_eval.py implemented; scoring and the eval loop unit-tested without Presidio.
  • test_eval_harness.py green; full suite green; no Presidio import at module load in evals/.
  • Gate scope and leakage handling match the recorded decisions; MRN/member/Rx reported, not gated.
