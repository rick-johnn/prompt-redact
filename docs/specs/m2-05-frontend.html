Spec M2-05 — Go front-end (public shell)

Fifth M2 spec (plan) and the piece deferred from the MVP. Completes the ADR 0001 sidecar: a Go binary is the public-facing service and forwards to the Python sidecar over loopback HTTP. Resolves the two deferred decisions — Go for the front-end, HTTP-over-loopback for the IPC.

Module: frontend/ (Go, stdlib only — no external deps). Tests: frontend/proxy_test.go.

Decisions resolved (2026-06-07)

  • Front-end language: Go. Single static binary, fast HTTP handling, trivial deploy — the conventional choice for a standalone network shell. (TypeScript would have been the pick for a Node-centric adopter.)
  • IPC: HTTP over loopback. The Python sidecar already speaks HTTP (FastAPI), so the front-end is a reverse proxy to 127.0.0.1 — no new interface, no codegen, easy to debug. gRPC would have meant adding a parallel gRPC server to the Python side. The IPC stays loopback-only (threat T11); the sidecar is never exposed publicly.

What it does (and deliberately doesn't)

The front-end is intentionally thin: a httputil.ReverseProxy to the sidecar, plus an edge body-size cap (413 before forwarding — threat T9, keeps oversized input off the NER pass) and no request-body logging (threat T2). The authoritative size enforcement, the redaction logic, and the error mapping all stay in the Python sidecar — the front-end does not re-implement them.

Honest note on value: because the sidecar already does the heavy lifting, the front-end's benefit is the topology (the model-bearing Python process is internal/loopback-only; a lean Go binary is what's exposed), not raw speed — the NER pass dominates latency, so the Go-vs-Python shell difference is negligible (consistent with ADR 0001's “the shell is not the bottleneck”).

Config (env)

PROMPT_REDACT_LISTEN          public listen address      (default ":8080")
PROMPT_REDACT_UPSTREAM        loopback sidecar base URL  (default "http://127.0.0.1:8000")
PROMPT_REDACT_MAX_BODY_BYTES  edge body-size cap         (default 1000000)

Public surface (frontend/)

  • newHandler(upstream, maxBytes) (http.Handler, error) — reverse proxy + edge cap; errors on a malformed upstream URL.
  • limitBody(next, maxBytes) — 413 on a declared body over the cap (maxBytes ≤ 0 disables).
  • main — reads env config and serves.

Test plan (frontend/proxy_test.go)

  • Requests forwarded to the upstream and its response proxied back (via an httptest upstream).
  • Oversized declared body → 413, upstream never reached.
  • /healthz forwarded.
  • Malformed upstream URL → constructor error.

Deployment

Two processes on one host/pod: the Go front-end (public) and the Python sidecar bound to loopback. Packaging them together (Dockerfile / compose) is M4; this spec delivers the front-end and its tests.

Definition of done

  • gofmt/go vet clean; go test ./... green.
  • Decisions (Go, HTTP-loopback) recorded in ADR 0001 and the M2 plan.
