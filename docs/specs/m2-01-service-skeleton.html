Spec M2-01 — Service skeleton + readiness

First implementation spec for M2. Stands up the FastAPI app and the GET /healthz readiness probe; the redaction endpoints arrive in M2-02. Per the M2 decisions, this is the Python service MVP (FastAPI); the Go/TS front-end is a later spec.

Package: prompt_redact_service/ — app.py (create_app factory + /healthz). Deps added to requirements.in: fastapi, uvicorn[standard], and httpx (test-only, for TestClient). Tests: tests/test_service_health.py.

Design

  • Eager analyzer build at startup. The lifespan handler builds (and warms) the analyzer before the service serves traffic. The analyzer's own docs warn its lazy first build isn't synchronized, so warming it up front avoids a concurrent-first-request race.
  • Injected analyzer provider. create_app(analyzer_provider=…) takes the builder as an argument; the default imports Presidio lazily and warms the model, while tests pass a fake provider. So importing prompt_redact_service needs FastAPI but not Presidio, and the whole skeleton is testable without the ML stack.
  • Up-but-unready on build failure. If the provider throws at startup, the service still starts but stays unready: /healthz returns 503, so a readiness probe fails (no traffic routed) instead of crash-looping. The failure is logged.
  • /healthz: 200 {"status":"ok"} once ready; 503 {"status":"unavailable"} otherwise.
  • ASGI entrypoint: module-level app = create_app() (real provider) for uvicorn prompt_redact_service.app:app.

Public surface

  • create_app(analyzer_provider=default_analyzer_provider) -> FastAPI
  • default_analyzer_provider() — builds + warms RedactionAnalyzer (lazy Presidio import).
  • GET /healthz.

Test plan (tests/test_service_health.py)

  • /healthz → 200 ok when the provider builds; → 503 unavailable when it raises.
  • Analyzer built eagerly exactly once at startup, not per request.
  • Service state holds the built analyzer and is ready; stays unready (analyzer None) after a failed build.
  • (All via injected fake providers + TestClient — no Presidio.)

Definition of done

  • prompt_redact_service/ app with /healthz; eager build via injected provider.
  • requirements.in updated (fastapi, uvicorn, httpx).
  • Tests green without Presidio; full suite green.
  • No Presidio import at package import time.
