Spec M2-04 — Multi-turn conversation test

Fourth M2 spec (plan) and the M2 exit criterion: a synthetic caller drives a multi-turn conversation through the service, round-tripping the token map exactly as a real chat app would, asserting cross-turn token stability and round-trip fidelity.

Test: tests/test_service_conversation.py. No app changes — this verifies the contract built in M2-01…03.

What it proves

  • Cross-turn token stability. The caller carries the token map out of each /redact response and back into the next /redact request; the same identifier (e.g. “John Doe”) keeps the same token ([PERSON_1]) across turns. This is the property that makes a redacted conversation coherent to a downstream LLM.
  • Fresh tokens for new identifiers. A new entity in turn 2 (“Jane Roe”) mints the next free token ([PERSON_2]) without colliding.
  • Round-trip fidelity. An LLM-style reply full of tokens is restored by /unredact using the accumulated map.

Approach

  • In-sandbox (no Presidio): a ScriptedAnalyzer returns deterministic detections (verified offsets) for each turn's text, so the test exercises the full service path — request validation, the flat-v1 wire contract, map merge across turns, replacement, and unredaction — and asserts exact redacted strings and maps. This proves the service contract end-to-end without the ML stack.
  • Presidio-backed variant (@pytest.mark.integration, skipped if unavailable): runs the real analyzer over a two-turn conversation and asserts the person's token is stable across turns and the second turn round-trips — without asserting exact offsets (which depend on the model).

Why TestClient, not docker-compose

The plan floated pulling compose forward to run this. It isn't needed: TestClient drives the real ASGI app in-process, which exercises the same request → redact → response → unredact contract. Compose / the curl smoke check belong with packaging in M4.

M2 (MVP) status

With M2-01…04 the Python-service MVP is contract-complete and unit-green: skeleton + readiness, both endpoints with the flat-v1 contract and error mapping, the size cap and scrubbed-500 hardening, and this multi-turn exit test. As with M1, the Presidio-backed confirmation (the integration variant + a manual curl round-trip) is the first action on a dev/CI machine with the model installed. The Go/TS front-end (M2-05) remains deferred pending the Go-vs-TS and IPC-protocol picks.

Definition of done

  • Multi-turn test green in-sandbox (fake analyzer): cross-turn stability, fresh-token minting, round-trip.
  • Presidio-backed variant present, skipped when the model is absent.
  • Full suite green.
