Spec M1-03 — Unredactor

Third implementation spec for M1. Implements unredact(text, token_map) -> text: the centralized token → original substitution. Pure-Python; reuses find_tokens and apply_replacements from Spec M1-01 — no Presidio dependency.

Module: prompt_redact_core/unredactor.py. New error: UnknownTokenError in errors.py. Maps to the plan's unredact() scope item and the test_unredactor.py test file.

The plan puts unredact in M1 (not M2) on purpose: the milestone exit criterion is a round trip over the corpus, and centralizing the substitution rules here keeps M2's /unredact a thin wrapper.

Algorithm

  1. Locate every [TYPE_N] in text with find_tokens (offsets, non-overlapping by construction).
  2. For each, look up its original in the map. Collect a Replacement(start, end, original) per known token; collect unknown tokens separately.
  3. If any token is unknown, raise UnknownTokenError (see below).
  4. Otherwise apply the replacements with apply_replacements — right-to-left, single pass.

Why single-pass matters (the re-substitution trap)

An original value can itself contain a token-shaped substring — e.g. the map holds {"[PERSON_1]": "agent [X_1] smith"}. A naive find-and-replace loop that re-scans after each substitution would then try to "unredact" the [X_1] it just inserted, corrupting the output (or raising on an unknown token).

apply_replacements avoids this by design: it computes all splice offsets against the original redacted text up front and applies them right-to-left, so inserted originals are never re-scanned. Reusing it here gives the unredactor that guarantee for free — and is the reason this spec renames the Replacement field from token to the direction-neutral replacement (it carries a minted token on the redact side and an original value here).

Resolved behaviour decisions

  • Unknown token → raise (strict). If text contains a [TYPE_N] that is not a key in the map, unredact raises UnknownTokenError rather than leaving it in place or guessing. In a correct round trip this cannot happen: the T5 guard (Spec M1-02) ensures the only token shapes in redacted text are ones we minted into the map. An unknown token therefore signals a corrupted or mismatched map, which a redaction tool should surface loudly, not paper over. (Decision 2026-06-05; revisit if a lenient/partial-unredact mode is ever needed.)
  • No full-map validation. unredact only reads the map by the keys present in text; it does not validate every map entry the way assign_tokens does. Validation belongs on the redact (minting) side; the unredact path is a pure lookup.
  • Empty original is distinct from missing. Lookup uses is None, so a map entry whose value is the empty string is treated as known (substitutes to nothing), not as an unknown token. (In practice empty originals never enter the map — Spec M1-01 skips empty detections — but the distinction is made explicit.)
  • No tokens / empty text → passthrough. Text with no token shapes (including "") is returned unchanged.

Round-trip property (tested now, without Presidio)

The headline property is unredact(redact(text)) == text. redact needs Presidio and arrives in Spec M1-06, but the round trip can be proven at the unit level today by standing in for the analyzer: construct detections by hand → assign_tokens → apply_replacements to get redacted text + map → unredact → assert it equals the original. This exercises the full token pipeline (mint, replace, look up, restore) end-to-end with synthetic spans, so Spec M1-06 only has to add real detection on top.

Public surface (unredactor.py)

  • unredact(text, token_map) -> str — restore originals; raises UnknownTokenError on an unmapped token.

Error (errors.py)

  • UnknownTokenError(RedactError) — carries .matches (the unmapped TokenMatches) and a message naming the first offender and its offset.

Refactor in this spec

Replacement.token → Replacement.replacement (and the redact-side tests that read it). The field is the string spliced into the text; calling it replacement reads correctly whether it holds a token (redact) or an original (unredact), and lets both paths share apply_replacements without a misleading name. No behaviour change.

Test plan (tests/test_unredactor.py)

  • Basic: single token, multiple tokens, repeated token (same token twice → same original both times).
  • Passthrough: empty string; text with no tokens.
  • Single-pass safety: an original containing a token shape ("agent [X_1] smith") is inserted verbatim and not re-substituted.
  • Unknown token: raises UnknownTokenError; .matches names the offender(s); a partially-known text still raises (all-or-nothing).
  • Empty-valued mapping: a token mapping to "" substitutes to nothing rather than raising.
  • Round trip (no Presidio): detections → assign_tokens → apply_replacements → unredact reconstructs the original, including repeated identifiers and Unicode text.
